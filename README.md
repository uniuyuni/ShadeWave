# Platypus — 開発メモ

このリポジトリはローカルで開発・デバッグするための手順をまとめたものです。特に `libraw` / `libraw_enhanced` のビルドを `.pixi` 環境に統一する手順を記載します。

## 前提
- macOS（Apple Silicon を想定）
- Xcode Command Line Tools（clang, make 等）
- `git` が `/usr/bin/git` で利用可能であること
- `.pixi` 開発環境（プロジェクトルートに `.pixi` ディレクトリ）

## 主要ポイント
- `.pixi/libraw-install` をプロジェクト内で共通の LibRaw インストール先として使う。
- `libraw_enhanced` は `.pixi` の `LIBRAW_LOCAL_PREFIX` を参照してビルドするように調整済み。

## .pixi 環境でのビルド手順（推奨）
1. pixi 環境を用意・有効化（プロジェクト固有の方法で）

2. LibRaw をビルドして `.pixi/libraw-install` にインストール

```bash
cd /path/to/Platypus
# scripts を使うか、libraw_enhanced/pixi.toml の tasks を使って実行
./scripts/build_libraw_from_external.sh   # または
cd libraw_enhanced && pixi build-libraw
```

3. `libraw_enhanced` を .pixi 環境の Python でインストール（editable）

```bash
cd /path/to/Platypus/libraw_enhanced
export LIBRAW_LOCAL_PREFIX="$(pwd)/../.pixi/libraw-install"  # 確認用
. /.pixi/envs/default/bin/python -m pip install -e . --no-build-isolation
```

4. （オプション）拡張ビルド後、リンク先を確認

```bash
otool -L libraw_enhanced/libraw_enhanced/_core.cpython-*.so
# 出力に .pixi/libraw-install/lib/libraw.*.dylib が含まれていることを確認
```

## 単体の再現スクリプト
問題切り分けのために `repro_raw_geom.py` を使います。

```bash
export LIBRAW_LOCAL_PREFIX="$(pwd)/.pixi/libraw-install"
. /.pixi/envs/default/bin/python repro_raw_geom.py /path/to/sample.RAF
```

## よくある問題と対処
- 'tuple' object has no attribute 'img' エラー: `file_cache_system._task_callback` 側で共有メモリタプルを `ImageSet` に復元するパッチを適用済み。該当ファイル: `file_cache_system.py`。
- シェーダーが見つからない: ルートの `metal` が `libraw_enhanced/core/metal` を指すよう symlink を作成してください。

```bash
# 例: プロジェクトルートで
if [ -e metal ] && [ ! -L metal ]; then mv metal metal.backup; fi
ln -sfn libraw_enhanced/core/metal metal
```
